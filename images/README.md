# Prebuilt QEMU Images

ArmOS release images are published as compressed GitHub Release assets rather
than committed to Git history. This keeps normal clones small while providing
a build-free path for both reference QEMU targets.

## ArmOS 0.7.4

Download one archive from:

- [ARM32 QEMU Virt](https://github.com/dev-recon/ArmOS/releases/download/0.7.4/ArmOS-0.7.4-qemu-virt-arm32.tar.gz)
- [ARM64 QEMU Virt](https://github.com/dev-recon/ArmOS/releases/download/0.7.4/ArmOS-0.7.4-qemu-virt-arm64.tar.gz)
- [SHA-256 checksums](https://github.com/dev-recon/ArmOS/releases/download/0.7.4/SHA256SUMS)

Extract an archive from the repository root. It installs the matching kernel
and disk under the ignored local `build/images` directory:

```sh
tar -xzf ArmOS-0.7.4-qemu-virt-arm32.tar.gz
TARGET_ARCH=arm32 TARGET_PLATFORM=qemu-virt ./boot.sh
```

For ARM64:

```sh
tar -xzf ArmOS-0.7.4-qemu-virt-arm64.tar.gz
TARGET_ARCH=arm64 TARGET_PLATFORM=qemu-virt ./boot.sh
```

Only QEMU itself is required to boot these artifacts. QEMU 10.0.2 is the
reference version; host installation details remain in
[`INSTALLATION_macos.md`](../INSTALLATION_macos.md) and
[`INSTALLATION_linux.md`](../INSTALLATION_linux.md).

The archives are refreshed only for tagged releases. Development builds
continue to use the local `build/images` tree generated by `build.sh`.

## Release hygiene

Before publication, both root filesystems and archive manifests are checked
for:

- runtime Wi-Fi profiles and credentials;
- host usernames and absolute build paths;
- crash dumps, shell histories, and recovery notes;
- Finder metadata, AppleDouble files, and other hidden macOS artifacts.

Only the public example profile under `/etc/wifi.d` is retained. The ext2
filesystems are checked with `e2fsck`, and `SHA256SUMS` authenticates the
published archives.
